## 1. 产品概述

危大工程方案现场交底移动 App，面向工地一线安全员、施工员和班组长，用于危大工程专项方案的现场交底、签到确认和反馈上报。解决传统纸质交底效率低、记录难追溯、工人理解难验证的痛点。

- 目标用户：施工现场安全员、施工员、班组长及一线工人
- 核心价值：班前会快速交底 → 签到留痕 → 要点问答验证 → 现场问题反馈，形成完整闭环

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 安全员 | 选择角色进入 | 发起交底、补充现场照片、查看交底记录、处理反馈 |
| 施工员 | 选择角色进入 | 查看方案卡、参与交底签到、要点问答、提交反馈 |
| 班组长 | 选择角色进入 | 组织班组签到、要点问答、提交现场反馈 |

### 2.2 功能模块

1. **方案卡模块**：项目选择 → 危大工程类型选择 → 方案详情卡片（关键控制点、施工顺序、验收节点、禁止事项）
2. **交底签到模块**：人员信息确认（姓名、工种、身份证）→ 电子签名 → 安全员补充照片 → 交底记录生成
3. **要点问答模块**：系统从方案抽取题目 → 工人答题 → 显示通过/需重新学习
4. **现场反馈模块**：拍照 + 位置描述 → 提交给技术负责人 → 反馈记录列表

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：工业橙（#F97316）作为强调色，搭配深灰（#1C1917）和暖白（#FAFAF9）
- **警示色**：禁止事项使用红色（#EF4444），通过/安全使用绿色（#22C55E）
- **按钮风格**：圆角矩形（rounded-xl），主按钮填充橙色，次按钮描边
- **字体**：标题使用 Noto Sans SC Bold，正文使用 Noto Sans SC Regular
- **布局风格**：移动端底部标签导航，卡片式布局，信息层级清晰
- **图标风格**：Lucide 线性图标，搭配简洁文字标签
- **整体风格**：工业实用主义，清晰高效，适合工地户外强光环境下阅读

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 角色选择页 | 角色卡片 | 三张角色卡片纵向排列，图标+角色名+简述，点击高亮选中 |
| 项目选择页 | 搜索栏+项目列表 | 顶部搜索框，下方项目卡片列表，显示项目名+地址+状态标签 |
| 工程类型页 | 类型网格 | 2列网格卡片，每张含图标+类型名，悬挑脚手架/暗挖/深基坑等 |
| 方案详情页 | 标签切换 | 顶部方案名称，四个Tab切换（控制点/施工顺序/验收/禁止），内容区滚动 |
| 交底签到页 | 表单+签名板 | 上部人员信息表单，中部签名画布，下部已签到列表 |
| 要点问答页 | 答题卡片 | 顶部进度条，中部题目卡片+选项，底部提交按钮 |
| 要点问答页 | 结果页 | 大号通过/未通过图标，错题回顾，重新学习按钮 |
| 现场反馈页 | 表单+记录 | 上部反馈表单（照片+文字），下部历史反馈时间线 |

### 4.3 响应式设计

- 移动端优先设计（375px-428px 基准宽度）
- 底部固定标签栏导航（方案卡/交底签到/反馈记录三个Tab）
- 触控优化：按钮最小点击区域44px，表单输入框高度48px
- 户外可读性：高对比度配色，关键信息大字号

### 4.4 数据说明

- 所有数据使用前端 Mock 数据，无需后端服务
- 方案数据、题目数据、签到记录、反馈记录均存储在 Zustand 状态管理中
